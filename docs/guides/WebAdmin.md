# Web Admin UI Documentation

## Overview
The MailRaven Web Admin UI is a React-based single-page application (SPA) built with Vite, TypeScript, Tailwind CSS, and Shadcn/UI. It provides a graphical interface for administrators to manage the MailRaven server.

## Tech Stack
- **Framework**: React 19 + Vite
- **Language**: TypeScript
- **Styling**: Tailwind CSS v4 + PostCSS
- **Components**: Shadcn/UI (Radix UI + Tailwind)
- **Routing**: React Router v7
- **HTTP Client**: Axios

## Project Structure
```
client/
├── src/
│   ├── components/         # Reusable UI components
│   │   ├── ui/             # Shadcn primitives (Button, Input, etc.)
│   │   ├── users/          # User-specific components (Dialogs, etc.)
│   │   └── ProtectedRoute.tsx
│   ├── contexts/           # React Contexts (AuthContext)
│   ├── layout/             # Layout components (MainLayout)
│   ├── lib/                # Utilities and API clients (api.ts, utils.ts)
│   ├── pages/              # Page components (Login, Dashboard, Users)
│   ├── App.tsx             # Main App component with Routing
│   ├── main.tsx            # Entry point
│   └── index.css           # Global styles and Tailwind imports
├── components.json         # Shadcn configuration
├── package.json            # Dependencies and scripts
└── vite.config.ts          # Vite configuration
```

## Setup & Running

1. **Install Dependencies**:
   ```bash
   cd client
   npm install
   ```

2. **Development Server**:
   ```bash
   npm run dev
   ```
   Access at `http://localhost:5173`.

3. **Production Build**:
   ```bash
   npm run build
   ```
   Output is generated in `client/dist`.

## Features

### 1. Authentication
- JWT-based login flow.
- Tokens stored in `localStorage`.
- Protected routes redirect unauthenticated users to `/login`.

### 2. Dashboard
- Displays system statistics:
  - Uptime
  - Total Users
  - Memory Usage
  - Goroutines
- Widgets update on mount.

### 3. User Management
- List all users.
- Create new users (with Email, Password, Role).
- Delete users.
- Role management (Admin/User).

### 4. Domain Management (In Progress)
- List hosted domains.
- Add new domains (Keys/DKIM generated by backend).
- Remove domains.

## API Integration
The frontend communicates with the backend via the `/api/v1` endpoints. CORS is configured on the backend to allow requests from the frontend development server.

See `client/src/lib/api.ts` for the API client implementation.
